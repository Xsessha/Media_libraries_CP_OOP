# Курсова робота: Менеджер медіатеки

**Тема:** Менеджер медіатеки (плейлисти, рейтинг, історія)  
**Дисципліна:** Об'єктно-орієнтоване програмування
## Функціональні можливості
Система надає користувачеві наступний базовий функціонал:
* **Управління колекцією медіафайлів:** перегляд загального каталогу, перегляд інформації про треки (назва, виконавець, жанр).
* **Робота з плейлистами:** створення персональних плейлистів, додавання та видалення треків.
* **Автоматизація списків:** автоматична генерація списку відтворення "Weekly Mix" на основі наявної бази медіафайлів.
* **Взаємодія з контентом:** вбудований інтерфейс плеєра для відтворення та перемикання треків у рамках обраного плейлиста.
* **Система оцінювання:** можливість виставляти рейтинги медіафайлам.
## Основні сутності
* **MediaItem:** Базова сутність медіаоб'єкта, що зберігає дані про назву, виконавця, жанр, тривалість, кількість відтворень (`PlayCount`) та розраховує середній рейтинг (`AverageRating`).
* **Playlist та PlaylistTrack:** Система дозволяє об'єднувати медіафайли у списки відтворення. Допоміжний проміжний клас `PlaylistTrack` забезпечує можливість додавання треків та фіксує порядок їх відтворення у конкретному списку.
* **User:** Описує користувача системи, зберігає його налаштування та містить колекції для індивідуальної взаємодії з медіатекою.
* **Rating та FavoriteMedia:** Механізми, що дозволяють користувачам виставляти оцінки трекам (формуючи загальний рейтинг системи) та додавати композиції до персонального списку улюблених пісень.
* **PlaybackHistory:** Клас, що фіксує активність користувачів (дату відтворення медіафайлу чи плейлиста), дозволяючи переглядати історію прослуховування та швидко повертатися до раніше відтвореного контенту.
